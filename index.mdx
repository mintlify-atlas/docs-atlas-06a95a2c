## Overview

Periodico Backend is a REST API built with **FastAPI** and **MySQL** that powers a digital newspaper platform. It handles the full lifecycle of news content — from writing and publishing articles to serving media assets and managing user accounts.

## Key features

- **Compressed post bodies** — Article content is stored as zlib-compressed JSON and decompressed on read, reducing database storage.
- **Section-based routing** — Posts are accessible by section name and URL slug: `GET /posts/{section}/{slug}`.
- **Role-based access control** — Admin endpoints are protected by a `requireAdmin` dependency that checks the JWT payload.
- **Media storage** — Images (JPEG, PNG, WebP, GIF up to 10 MB) and videos (MP4, WebM up to 50 MB) are stored as binary blobs in MySQL with streaming delivery.
- **Image compression** — Uploaded images are automatically compressed before storage to reduce size.
- **PDF to HTML** — Convert PDF files to HTML for use in article content via a dedicated admin endpoint.
- **CORS enabled** — The API accepts cross-origin requests from any origin, suitable for integration with a frontend application.
