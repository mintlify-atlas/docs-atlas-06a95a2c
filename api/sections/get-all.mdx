---
title: "Get all sections"
description: "Returns a list of all sections available in the platform."
---

Returns every section record in the database. Sections are the top-level categories that organize posts on the platform (e.g., Politics, Sports, Technology).

<Note>This is a public endpoint. No authentication is required.</Note>

## Request

**GET** `/sections/get-all`

This endpoint accepts no parameters.

## Example request

<CodeGroup>

```bash cURL
curl --request GET \
  --url https://your-api-domain.com/sections/get-all
```

```python Python
import requests

response = requests.get("https://your-api-domain.com/sections/get-all")
data = response.json()
```

```javascript JavaScript
const response = await fetch("https://your-api-domain.com/sections/get-all");
const data = await response.json();
```

</CodeGroup>

## Response

Returns an array of section objects. Returns an empty array if no sections have been created.

<ResponseField name="" type="object[]">
  Array of section objects.

  <Expandable title="Section object properties" defaultOpen={true}>
    <ResponseField name="id" type="number" required>
      Unique identifier for the section.
    </ResponseField>

    <ResponseField name="name" type="string" required>
      Display name of the section. This value is used as the `section` parameter in post lookup endpoints.
    </ResponseField>
  </Expandable>
</ResponseField>

## Example response

```json 200
[
  {
    "id": 1,
    "name": "Politics"
  },
  {
    "id": 2,
    "name": "Sports"
  },
  {
    "id": 3,
    "name": "Technology"
  }
]
```

## Error responses

This endpoint does not produce client errors under normal operation.

| Status | Description |
|--------|-------------|
| `500`  | Internal server error. The database may be unreachable. |
