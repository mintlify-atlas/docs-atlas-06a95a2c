---
title: "Get section by ID"
description: "Returns a single section by its unique numeric ID."
---

Looks up a section by its primary key.

<Note>This is a public endpoint. No authentication is required.</Note>

<Warning>
  This endpoint returns `null` when no section matches the given ID. It does not return a `404` status code. Check whether the response body is `null` before using the data.
</Warning>

## Request

**GET** `/sections/get-by-id`

## Parameters

<ParamField query="id" type="number" required>
  The unique numeric ID of the section to retrieve.
</ParamField>

## Example request

<CodeGroup>

```bash cURL
curl --request GET \
  --url "https://your-api-domain.com/sections/get-by-id?id=1"
```

```python Python
import requests

response = requests.get(
    "https://your-api-domain.com/sections/get-by-id",
    params={"id": 1}
)
data = response.json()
```

```javascript JavaScript
const response = await fetch("https://your-api-domain.com/sections/get-by-id?id=1");
const data = await response.json();
```

</CodeGroup>

## Response

Returns a single section object if found, or `null` if no section exists with the given ID.

<ResponseField name="id" type="number" required>
  Unique identifier for the section.
</ResponseField>

<ResponseField name="name" type="string" required>
  Display name of the section.
</ResponseField>

## Example responses

<CodeGroup>

```json 200 (found)
{
  "id": 1,
  "name": "Politics"
}
```

```json 200 (not found)
null
```

</CodeGroup>

## Error responses

| Status | Description |
|--------|-------------|
| `422`  | Validation error. The `id` parameter was not provided or is not a valid integer. |
| `500`  | Internal server error. The database may be unreachable. |
