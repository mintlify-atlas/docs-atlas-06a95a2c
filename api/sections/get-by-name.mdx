---
title: "Get section by name"
description: "Returns a single section by its exact display name."
---

Looks up a section by its `name` field. This is useful for resolving a section name to its numeric ID.

<Note>This is a public endpoint. No authentication is required.</Note>

<Warning>
  This endpoint returns `null` when no section matches the given name. It does not return a `404` status code. Section name matching is case-sensitive.
</Warning>

## Request

**GET** `/sections/get-by-name`

## Parameters

<ParamField query="name" type="string" required>
  The exact display name of the section to look up. Matching is case-sensitive.
</ParamField>

## Example request

<CodeGroup>

```bash cURL
curl --request GET \
  --url "https://your-api-domain.com/sections/get-by-name?name=Politics"
```

```python Python
import requests

response = requests.get(
    "https://your-api-domain.com/sections/get-by-name",
    params={"name": "Politics"}
)
data = response.json()
```

```javascript JavaScript
const response = await fetch(
  "https://your-api-domain.com/sections/get-by-name?name=Politics"
);
const data = await response.json();
```

</CodeGroup>

## Response

Returns a single section object if found, or `null` if no section has the given name.

<ResponseField name="id" type="number" required>
  Unique identifier for the section.
</ResponseField>

<ResponseField name="name" type="string" required>
  Display name of the section.
</ResponseField>

## Example responses

<CodeGroup>

```json 200 (found)
{
  "id": 1,
  "name": "Politics"
}
```

```json 200 (not found)
null
```

</CodeGroup>

## Error responses

| Status | Description |
|--------|-------------|
| `422`  | Validation error. The `name` parameter was not provided. |
| `500`  | Internal server error. The database may be unreachable. |

<Tip>
  To get a full list of valid section names, use the [GET /sections/get-all](/api/sections/get-all) endpoint.
</Tip>
