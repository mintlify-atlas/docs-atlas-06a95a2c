---
title: "Get current user"
description: "Return the profile of the currently authenticated user based on the JWT token."
---

`GET /users/me`

Returns the full profile of the authenticated user decoded from the provided Bearer token.

<Warning>
  This endpoint requires authentication. Requests without a valid token will be rejected with `401`.
</Warning>

## Authentication

<ParamField header="Authorization" type="string" required>
  Bearer token obtained from the [login](/api/users/login) endpoint.

  Format: `Bearer <token>`
</ParamField>

## Example

<CodeGroup>

```bash cURL
curl --request GET \
  --url 'https://api.example.com/users/me' \
  --header 'Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...'
```

</CodeGroup>

## Response

<ResponseField name="id" type="number" required>
  Unique numeric identifier for the user.
</ResponseField>

<ResponseField name="username" type="string" required>
  Display name of the user.
</ResponseField>

<ResponseField name="gmail" type="string" required>
  Gmail address of the user.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of when the account was created.
</ResponseField>

<ResponseField name="admin" type="boolean" required>
  Whether the user has administrator privileges.
</ResponseField>

```json 200 Success
{
  "id": 1,
  "username": "johndoe",
  "gmail": "john@example.com",
  "created_at": "2024-01-15T10:30:00",
  "admin": false
}
```

## Error codes

| Code | Cause |
| ---- | ----- |
| `401` | Token is missing, malformed, or has expired. |
