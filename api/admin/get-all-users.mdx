---
title: "Get all users"
description: "Retrieve a list of every registered user on the platform."
---

<Warning>
  Admin authentication is required. Include a valid `Authorization: Bearer <token>` header with admin privileges.
</Warning>

## Endpoint

```
GET /admin/users
```

## Parameters

<ParamField header="Authorization" type="string" required>
  Bearer token for admin authentication. Format: `Bearer <token>`
</ParamField>

This endpoint takes no query or path parameters.

## Request example

```bash cURL
curl --request GET \
  --url 'https://api.example.com/admin/users' \
  --header 'Authorization: Bearer <your-admin-token>'
```

## Response

**200 OK**

Returns an array of user objects.

<ResponseField name="id" type="number">
  Unique identifier for the user.
</ResponseField>

<ResponseField name="username" type="string">
  The user's display name.
</ResponseField>

<ResponseField name="gmail" type="string">
  The user's Google email address used for authentication.
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of when the account was created.
</ResponseField>

<ResponseField name="admin" type="boolean">
  Whether the user has admin privileges.
</ResponseField>

```json 200
[
  {
    "id": 1,
    "username": "johndoe",
    "gmail": "john@example.com",
    "created_at": "2024-01-15T10:30:00",
    "admin": false
  },
  {
    "id": 2,
    "username": "janedoe",
    "gmail": "jane@example.com",
    "created_at": "2024-02-20T08:15:00",
    "admin": true
  }
]
```

## Error codes

| Status | Description |
|--------|-------------|
| `401`  | Missing or invalid authorization token. |
| `403`  | Token is valid but the user does not have admin privileges. |
