---
title: "Delete post"
description: "Permanently delete a post by its ID."
---

<Warning>
  Admin authentication is required. Include a valid `Authorization: Bearer <token>` header with admin privileges.
</Warning>

<Warning>
  This operation is irreversible. The post and all associated data cannot be recovered after deletion.
</Warning>

## Endpoint

```
DELETE /admin/delete-post
```

## Parameters

<ParamField query="id" type="number" required>
  The unique ID of the post to delete.
</ParamField>

<ParamField header="Authorization" type="string" required>
  Bearer token for admin authentication. Format: `Bearer <token>`
</ParamField>

## Request example

```bash cURL
curl --request DELETE \
  --url 'https://api.example.com/admin/delete-post?id=42' \
  --header 'Authorization: Bearer <your-admin-token>'
```

## Response

**200 OK**

<ResponseField name="status" type="string">
  Always `"ok"` on success.
</ResponseField>

<ResponseField name="message" type="string">
  Human-readable confirmation message.
</ResponseField>

```json 200
{
  "status": "ok",
  "message": "Post borrado correctamente"
}
```

## Error codes

| Status | Description |
|--------|-------------|
| `401`  | Missing or invalid authorization token. |
| `403`  | Token is valid but the user does not have admin privileges. |
