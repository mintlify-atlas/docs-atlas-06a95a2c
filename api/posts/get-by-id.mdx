---
title: "Get post by ID"
description: "Returns a single post by its unique numeric ID."
---

Looks up a post by its primary key. The `body` field is stored compressed and is automatically decompressed into a structured JSON object before being returned.

<Note>This is a public endpoint. No authentication is required.</Note>

<Warning>
  This endpoint returns `null` when no post matches the given ID. It does not return a `404` status code. Check whether the response body is `null` before using the data.
</Warning>

## Request

**GET** `/posts/get-by-id`

## Parameters

<ParamField query="id" type="number" required>
  The unique numeric ID of the post to retrieve.
</ParamField>

## Example request

<CodeGroup>

```bash cURL
curl --request GET \
  --url "https://your-api-domain.com/posts/get-by-id?id=1"
```

```python Python
import requests

response = requests.get(
    "https://your-api-domain.com/posts/get-by-id",
    params={"id": 1}
)
data = response.json()
```

```javascript JavaScript
const response = await fetch("https://your-api-domain.com/posts/get-by-id?id=1");
const data = await response.json();
```

</CodeGroup>

## Response

Returns a single post object if found, or `null` if no post exists with the given ID.

<ResponseField name="id" type="number" required>
  Unique identifier for the post.
</ResponseField>

<ResponseField name="title" type="string" required>
  The headline or title of the post.
</ResponseField>

<ResponseField name="slug" type="string" required>
  URL-friendly identifier derived from the title.
</ResponseField>

<ResponseField name="body" type="object" required>
  Decompressed post content.

  <Expandable title="body properties">
    <ResponseField name="subtitulo" type="string">
      A brief subtitle or summary for the article.
    </ResponseField>

    <ResponseField name="contenido_html" type="string">
      Full article content as an HTML string.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="user_id" type="number" required>
  Identifier of the user (author) who created the post.
</ResponseField>

<ResponseField name="section_id" type="number" required>
  Identifier of the section this post belongs to.
</ResponseField>

<ResponseField name="published" type="number" required>
  Publication status. `1` indicates the post is published, `0` means it is unpublished.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of when the post was created.
</ResponseField>

## Example responses

```json 200 (found)
{

  "slug": "breaking-news-example-article",

  "user_id": 1,
```

```json 200 (not found)
null
```

</CodeGroup>

## Error responses

| Status | Description |
|--------|-------------|
| `422`  | Validation error. The `id` parameter was not provided or is not a valid integer. |
| `500`  | Internal server error. The database may be unreachable. |
