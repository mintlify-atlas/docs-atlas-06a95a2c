---
title: "Get post by slug"
description: "Returns a single post identified by its section name and slug."
---

Looks up a specific post using the combination of a section name and a URL slug. Both path parameters are required. The `body` field is stored compressed and is automatically decompressed before being returned.

<Note>This is a public endpoint. No authentication is required.</Note>

<Tip>
  The slug-and-section combination is unique, making this the canonical way to link to individual articles from front-end clients.
</Tip>

## Request

**GET** `/posts/{section}/{slug}`

## Parameters

<ParamField path="section" type="string" required>
  The name of the section the post belongs to. Must match an existing section name exactly.
</ParamField>

<ParamField path="slug" type="string" required>
  The URL slug of the post. Slugs are unique within a given section.
</ParamField>

## Example request

<CodeGroup>

```bash cURL
curl --request GET \
  --url "https://your-api-domain.com/posts/Politics/senate-passes-new-budget"
```

```python Python
import requests

response = requests.get(
    "https://your-api-domain.com/posts/Politics/senate-passes-new-budget"
)
data = response.json()
```

```javascript JavaScript
const response = await fetch(
  "https://your-api-domain.com/posts/Politics/senate-passes-new-budget"
);
const data = await response.json();
```

</CodeGroup>

## Response

Returns a single post object.

<ResponseField name="id" type="number" required>
  Unique identifier for the post.
</ResponseField>

<ResponseField name="title" type="string" required>
  The headline or title of the post.
</ResponseField>

<ResponseField name="slug" type="string" required>
  URL-friendly identifier for the post within its section.
</ResponseField>

<ResponseField name="body" type="object" required>
  Decompressed post content.

  <Expandable title="body properties">
    <ResponseField name="subtitulo" type="string">
      A brief subtitle or summary for the article.
    </ResponseField>

    <ResponseField name="contenido_html" type="string">
      Full article content as an HTML string.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="user_id" type="number" required>
  Identifier of the user (author) who created the post.
</ResponseField>

<ResponseField name="section_id" type="number" required>
  Identifier of the section this post belongs to.
</ResponseField>

<ResponseField name="published" type="number" required>
  Publication status. `1` indicates the post is published, `0` means unpublished.
</ResponseField>

<ResponseField name="created_at" type="string" required>
  ISO 8601 timestamp of when the post was created.
</ResponseField>

## Example response

```json 200
{
  "id": 4,
  "title": "Senate Passes New Budget",
  "slug": "senate-passes-new-budget",
  "body": {
    "subtitulo": "A landmark decision for fiscal policy",
    "contenido_html": "<p>The senate voted 60-40 in favor...</p>"
  },
  "user_id": 2,
  "section_id": 1,
  "published": 1,
  "created_at": "2024-01-15T09:00:00"
}
```

## Error responses

| Status | Description |
|--------|-------------|
| `404`  | The specified section name does not exist, or no post with the given slug exists in that section. |
| `500`  | Internal server error. The database may be unreachable. |
