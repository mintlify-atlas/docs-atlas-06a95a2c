---
title: "Get all posts"
description: "Returns all published posts ordered by creation date, newest first."
---

Returns the complete list of posts from the database, ordered by `created_at` descending. The `body` field is stored compressed and is automatically decompressed into a structured JSON object before being returned.

<Note>This is a public endpoint. No authentication is required.</Note>

## Request

**GET** `/posts/get-all`

This endpoint accepts no parameters.

## Example request

<CodeGroup>

```bash cURL
curl --request GET \
  --url https://your-api-domain.com/posts/get-all
```

```python Python
import requests

response = requests.get("https://your-api-domain.com/posts/get-all")
data = response.json()
```

```javascript JavaScript
const response = await fetch("https://your-api-domain.com/posts/get-all");
const data = await response.json();
```

</CodeGroup>

## Response

Returns an array of post objects. An empty array is returned when no posts exist.

<ResponseField name="" type="object[]">
  Array of post objects.

  <Expandable title="Post object properties" defaultOpen={true}>
    <ResponseField name="id" type="number" required>
      Unique identifier for the post.
    </ResponseField>

    <ResponseField name="title" type="string" required>
      The headline or title of the post.
    </ResponseField>

    <ResponseField name="slug" type="string" required>
      URL-friendly identifier derived from the title. Used in conjunction with the section name to identify a post.
    </ResponseField>

    <ResponseField name="body" type="object" required>
      Decompressed post content. Originally stored as compressed binary in the database.

      <Expandable title="body properties">
        <ResponseField name="subtitulo" type="string">
          A brief subtitle or summary for the article.
        </ResponseField>

        <ResponseField name="contenido_html" type="string">
          Full article content as an HTML string.
        </ResponseField>
      </Expandable>
    </ResponseField>

    <ResponseField name="user_id" type="number" required>
      Identifier of the user (author) who created the post.
    </ResponseField>

    <ResponseField name="section_id" type="number" required>
      Identifier of the section this post belongs to.
    </ResponseField>

    <ResponseField name="published" type="number" required>
      Publication status. `1` indicates the post is published, `0` means it is unpublished.
    </ResponseField>

    <ResponseField name="created_at" type="string" required>
      ISO 8601 timestamp of when the post was created.
    </ResponseField>
  </Expandable>
</ResponseField>

## Error responses

This endpoint does not produce client errors under normal operation. Server-side errors may occur if the database is unavailable.

| Status | Description |
|--------|-------------|
| `500`  | Internal server error. The database may be unreachable. |
